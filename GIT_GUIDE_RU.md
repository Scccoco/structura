# Шпаргалка по Git для "Вайбкодинга"

Просто копируй и вставляй эти команды в командную строку (cmd).

## 1. Как сохранить ВСЁ (самый частый вариант)
Например, когда мы обновили версию или поменяли настройки, и нужно просто "засэйвить" всё состояние проекта.

```cmd
cd c:\structura
git add .
git commit -m "обновление настроек"
git push
```
*(Вместо `"обновление настроек"` можно писать любой текст в кавычках)*

## 2. Как сохранить ТОЛЬКО то, что изменилось
На самом деле команда `git add .` (с точкой) сама находит всё, что изменилось. Так что используй вариант из пункта 1, он универсальный.

Но если ты хочешь проверить статус перед сохранением:
```cmd
git status
```
*Красным* — то, что изменилось, но еще не готово к сохранению.
*Зеленым* — то, что уже готово к коммиту.

## 3. Как обновить НАСЕРВЕРЕ (VPS)
Когда ты уже всё запушил (сделал пункт 1) и коммит улетел на GitHub, иди на сервер.
Подключись к серверу через терминал (SSH) и в папке проекта выполни:

```bash
# 1. Заходим в папку проекта
cd /root/structura

# 2. Скачиваем обновления с GitHub
git pull

# 3. Перезапускаем контейнеры (чтобы применилась новая версия)
cd docker
docker-compose -f docker-compose.base.yml -f docker-compose.prod.yml up -d
```
*(`-d` означает "в фоне", чтобы терминал не завис)*

---
### Если что-то пошло не так на сервере
Если `git pull` ругается, можно сделать "сброс" локальных изменений на сервере (осторожно, это удалит то, что ты менял на сервере руками, если менял):
```bash
git reset --hard
git pull
```
