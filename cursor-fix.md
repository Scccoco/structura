# Решение проблемы с терминалом в Cursor

## Проблема
Перед командами добавляются символы "qс" (кириллица), что мешает выполнению команд.

## Решения

### Вариант 1: Перезапуск терминала в Cursor
1. Закройте текущий терминал (иконка корзины)
2. Откройте новый терминал: `Ctrl + Shift + `` (обратная кавычка)
3. Или через меню: Terminal → New Terminal

### Вариант 2: Изменить тип терминала
1. Откройте настройки Cursor: `Ctrl + ,`
2. Найдите `terminal.integrated.defaultProfile.windows`
3. Измените на `Command Prompt` вместо `PowerShell`
4. Перезапустите терминал

### Вариант 3: Использовать Command Prompt
1. В Cursor: Terminal → New Terminal
2. В правом верхнем углу терминала выберите "Select Default Profile"
3. Выберите "Command Prompt"
4. Откройте новый терминал

### Вариант 4: Запуск от администратора
1. Закройте Cursor
2. Правой кнопкой на ярлык Cursor → "Запуск от имени администратора"
3. Откройте проект заново

### Вариант 5: Настройка кодировки PowerShell
Добавьте в начало `start.ps1`:
```powershell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
$OutputEncoding = [System.Text.Encoding]::UTF8
```

### Вариант 6: Использовать внешний терминал
1. Откройте PowerShell или CMD отдельно (не в Cursor)
2. Перейдите в директорию проекта: `cd C:\structura`
3. Запустите команды оттуда

## Проверка
После применения решения попробуйте:
```powershell
docker --version
```

Если команда выполняется без ошибок - проблема решена!
